# Security & License Enforcement

## Report a Security Vulnerability

If you discover a security vulnerability in ariaflow, please report it
privately via [GitHub Security Advisories](https://github.com/bonomani/ariaflow/security/advisories/new).

Do NOT open a public issue for security vulnerabilities.

## Report a License Violation

If you believe the [LICENSE](./LICENSE) has been violated:

### 1. Code Redistribution

If ariaflow source code or binaries have been redistributed without
authorization:

- **GitHub:** File a [DMCA takedown request](https://docs.github.com/en/site-policy/content-removal-policies/dmca-takedown-policy)
- **PyPI:** Report to [PyPI support](https://pypi.org/help/#feedback)
- **Other platforms:** Contact the copyright holder

### 2. Unauthorized AI Training

If ariaflow source code has been used to train an AI model without
consent (see LICENSE Section 3):

- Open a [GitHub Issue](https://github.com/bonomani/ariaflow/issues)
  with evidence
- Contact the copyright holder directly

### 3. Contact

- **GitHub Issues:** https://github.com/bonomani/ariaflow/issues
- **Copyright holder:** bonomani (via GitHub profile)
